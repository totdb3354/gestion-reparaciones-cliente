# Gestión de Clientes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Añadir un catálogo de clientes y vincularlo a los teléfonos, para indicar el cliente al asignar reparaciones y verlo/editarlo en las tablas, con logs de auditoría y bloqueo optimista.

**Architecture:** Cliente-servidor Java. Cliente JavaFX (DAOs vía `ApiClient` REST). Servidor Spring Boot + `JdbcTemplate` sobre MariaDB. Nueva entidad `Cliente` (catálogo) y FK `Telefono.ID_CLI`. El cliente es atributo del IMEI (vive en `Telefono`). Se sigue el patrón existente de CRUD (Técnicos/Proveedores), logs (`LogDAO`) y concurrencia optimista (`UPDATED_AT` + `409`/`StaleDataException`).

**Tech Stack:** Java 17+, Spring Boot, JdbcTemplate, MariaDB, JavaFX, JUnit 5, Mockito, Maven.

## Global Constraints

- Diseño de referencia: `docs/superpowers/specs/2026-06-23-clientes-design.md`.
- Rama de trabajo: `feature/clientes` (repo `ProgramaReparaciones`). **No** hacer merge ni push sin confirmación del usuario.
- Commits: mensajes en español, estilo convencional (`feat(...)`, `test(...)`, `chore(...)`). **No** añadir `Co-Authored-By`.
- Idioma del código y la UI: español (nombres de variables/mensajes como el resto del repo).
- Permisos: gestión/asignación/edición de cliente y edición de observación de teléfono = **SUPERTECNICO y ADMIN**. Lectura del catálogo = cualquier rol autenticado. TECNICO ve la pantalla de clientes en solo lectura.
- Concurrencia: toda edición de `Cliente` y `Telefono` usa `UPDATED_AT`; el servidor lanza `ResponseStatusException(HttpStatus.CONFLICT, ...)` si el `UPDATE ... WHERE ... AND UPDATED_AT = ?` afecta 0 filas; comparar con `Timestamp.valueOf(updatedAt.truncatedTo(ChronoUnit.SECONDS))`. El cliente recibe `409` → `StaleDataException` (mapeo ya existente en `ApiClient`).
- Logs: cada escritura nueva llama `logDao.insertar(principal.getIdUsu(), ACCION, detalle)`.
- Enfoque de tests (acorde al repo): TDD estricto en **modelos POJO** (hay tests de modelo existentes: `TecnicoTest`, `ReparacionResumenTest`). DAOs/controladores/UI: no hay arnés de test (ni TestFX); se verifican con `mvn -q compile` (compilación) y verificación manual descrita en cada tarea. No inventar tests de UI.
- Comandos Maven:
  - Servidor: `cd gestion-reparaciones-servidor && mvn -q -DskipTests compile` / `mvn -q test`
  - Cliente: `cd gestion-reparaciones-cliente && mvn -q -DskipTests compile` / `mvn -q test`

### Task 2.2: `ClienteDAO` (servidor)

**Files:**
- Create: `gestion-reparaciones-servidor/src/main/java/com/reparaciones/servidor/dao/ClienteDAO.java`

**Interfaces:**
- Consumes: tabla `Cliente`.
- Produces: `getAll()`, `getActivos()`, `insertar(nombre)`, `editar(idCli, nombre, updatedAt)`, `setActivo(idCli, activo, updatedAt)`, `tieneTelefonos(idCli)`, `getNombreById(idCli)`. `editar`/`setActivo` lanzan `409` si 0 filas.

- [ ] **Step 1: Crear la clase**

```java
package com.reparaciones.servidor.dao;

import com.reparaciones.servidor.model.Cliente;
import org.springframework.http.HttpStatus;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.jdbc.core.RowMapper;
import org.springframework.stereotype.Repository;
import org.springframework.web.server.ResponseStatusException;

import java.sql.Timestamp;
import java.time.LocalDateTime;
import java.time.temporal.ChronoUnit;
import java.util.List;

@Repository
public class ClienteDAO {

    private final JdbcTemplate jdbc;

    private static final RowMapper<Cliente> MAPPER = (rs, row) -> new Cliente(
            rs.getInt("ID_CLI"),
            rs.getString("NOMBRE"),
            rs.getBoolean("ACTIVO"),
            rs.getTimestamp("UPDATED_AT").toLocalDateTime());

    public ClienteDAO(JdbcTemplate jdbc) { this.jdbc = jdbc; }

    public List<Cliente> getAll() {
        return jdbc.query(
                "SELECT ID_CLI, NOMBRE, ACTIVO, UPDATED_AT FROM Cliente ORDER BY NOMBRE", MAPPER);
    }

    public List<Cliente> getActivos() {
        return jdbc.query(
                "SELECT ID_CLI, NOMBRE, ACTIVO, UPDATED_AT FROM Cliente WHERE ACTIVO = 1 ORDER BY NOMBRE",
                MAPPER);
    }

    public void insertar(String nombre) {
        jdbc.update("INSERT INTO Cliente (NOMBRE, ACTIVO) VALUES (?, 1)", nombre);
    }

    public void editar(int idCli, String nombre, LocalDateTime updatedAt) {
        int filas = jdbc.update(
                "UPDATE Cliente SET NOMBRE = ? WHERE ID_CLI = ? AND UPDATED_AT = ?",
                nombre, idCli, Timestamp.valueOf(updatedAt.truncatedTo(ChronoUnit.SECONDS)));
        if (filas == 0) {
            throw new ResponseStatusException(HttpStatus.CONFLICT, "Dato modificado por otro usuario");
        }
    }

    public void setActivo(int idCli, boolean activo, LocalDateTime updatedAt) {
        int filas = jdbc.update(
                "UPDATE Cliente SET ACTIVO = ? WHERE ID_CLI = ? AND UPDATED_AT = ?",
                activo ? 1 : 0, idCli, Timestamp.valueOf(updatedAt.truncatedTo(ChronoUnit.SECONDS)));
        if (filas == 0) {
```
